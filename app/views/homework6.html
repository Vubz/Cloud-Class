The Five Pillars of the Well-Architected Framework

Al momento de generar soluciones si se descuidan los 5 pilares de la arquitectura de software, se puede tener una solución no durable que puede colapsar con el tiempo.

Seguridad

Es la habilidad de proteger la información, sistemas y bienes a través de la implementación de estrategias y manejo de riesgos.

Principios:

Aplicar seguridad a todas las capas: Cada capa debe tener sus propias medidas de seguridad en lugar de los extremos. Habilitar rastreabilidad: Hacer log de todos los cambios de ambiente. Implementar el concepto de menor privilegio: Autorización adecuada para cada recurso. Enfoque en la seguridad del ambiente: Automatizar las mejores prácticas de seguridad: Mecanismos de software ayudan a una mejor escabilidad para una protección más efectiva.

Definición:

Las mejores prácticas para el manejo de seguridad son: Las Mejores practicas
  1. Identidad y acceso de acceso: Estas son las bases para manejar la autentificación autorizada pos solo los usuarios con el privilegio de revisar la información que les corresponde. En Amazon Web Services esto es manejado por AWS Identity u Access Management (IAM), donde se crean pólizas para administrar recursos, grupos o recursos.
  2. Controles detectives: Esta parte se refiere más al manejar de la información generada por el sistema, la lectura de log, eventos, y monitoreo que ayuda a la auditoria, análisis automatizados y alarmas. AWS CloudTrail logs, llamadas a AWS API , y Amazon CloudWatch proveen logs y métricas para el monitoreo con alarmas.
  3. Protección de infraestructura: Esto está basado en el control de metodologías para proteger la profundidad y la autentificación de varios factores, que son necesarios para tener las mejores practias. EnAWS se puede implementar la inspección de paquetes con herramientas nativas o externas.
  4. Protección de información: Las practicas más importantes para la protección de información son:
    1. Que los usuarios mantengan control de su información
    2. Encriptar la información y keys
    3. Log detallados del contenido importante
    4. Durabilidad de la arquitectura
    5. Versionamiento
    6. Mantener la información en una sola región hasta que sea necesario moverla.
  5. Respuesta a incidentes: Procesos para responder a errores, incidentes de seguridad y otros eventos que tengan un impacto en la seguridad del sistema. Logs detallados son la mejor herramienta para la prevención de estos incidentes y sus herramientas de monitoreo se pueden encontrar en AWS.

Confiabilidad

Es la habilidad del sistema para recuperarse de interrupciones de servicio, requiere de computación constante y dinámica para calcular la demanda y proveer el servicio necesario.

Principios:

  1. Exámenes de proceso de recuperación: Exámenes para asegurarse que el sistema funciona en ambientes diferentes.
  2. Recuperarse automáticamente después de un fallo: Monitoreo del sistema por indicadores clave para activar un límite.
  3. Escalabilidad horizontal para agregar disponibilidad de sistema: Remplazar un recurso grande por varios pequeños para garantizar disponibilidad.
  4. Evitar adivinar la capacidad: Evitar saturación.
  5. Manejar cambio en automatización: Cambios automatizados.
Definición:
  1. Fundaciones
  2. Manejo de cambios
  3. Manejo de errores

Fundaciones: Antes generar la infraestructura de cualquier sistema es necesario que haya confiabilidad como base del sistema. Con AWS la mayoría de la fundación ya se encuentra en pie, así que la pregunta es cómo manejar los servicios y pensar en la topología.

Manejo de cambio. Planear a futuro con la información de tus sistema permite brindar el mejor servicio, estos cambios y variables tienen que ser monitoreados de manera automática. Para el aumento de servidores para permitir más usuarios por ejemplo.

Manejo de fallo: El fallo es inevitable, el mejor interés es como responder a estar fallas y prevenir que sucedan otra vez. Además del monitoreo, respaldos de información son importantes para la respuesta, también exámenes de sistema para comprobar el funcionamiento.

Eficiencia de funcionamiento.

La mejor manera de utilizar las tecnologías para cumplir con los requisitos.

Principios

  1. Democratizar tecnologías avanzadas: Rentar nuevas tecnologías como un servicio en lugar de aprender a usarlas.
  2. Irse global en minutos: Hacer el sistema fácil de implementar en múltiples regiones con pocos clicks.
  3. Arquitecturas serverless: Rentar servidores elimina el mantenimiento dejando la necesidad solo de implementar.
  4. Experimentar seguido: Experimentar nuevas configuraciones y compatibilidad.
  5. Simpatia mecánica; Usar las tecnologías con las que te sientas comodo.

Definicion:

  1. Selección
  2. Revisión
  3. Monitoreo
  4. Intercambios

Selección:

La solución óptima para cualquier sistema particular está basado en el tipo de trabajo que se tiene. El tipo de arquitectura está basado en la optimización del sistema para brindar el trabajo deseado.
  1. Computación: la computación optima se basa en el diseño, patrones, etc, tu arquitectura debe tomar ventaja de la elasticidad que permite la arquitectura serverless.
  2. Guardado: Se basa en la frecuencia que se requiere accesar la infomacion, la cantidad de cambios a la que es sometida, si es dinámica, etc. En AWS existen muchos tipos de servicios para elegir tu guardado de datos ideal.
  3. Base de datos: Por lo general se elige de acuerdo a la organización en lugar de tener un aproche por datos.
  4. Network: Al estar virtualizado AWS ofrece múltiples servicios para acodar la network a tus necesidades.
Revisión: Revisión constante a los cambios y servicios de AWS para asegurarse que se tiene los mejores recursos para el servicio. Monitoreo: Usar los servicios de AWS para monitorear los logs y otras métricas. Intercambios: Entender las altas y bajas de los recursos que se seleccionaron para brindar el servicio óptimo.

Optimización de costos.

El proceso de refinamiento y mejora de sistema dentro del ciclo de vida, desde pruebas de concepto, hasta pruebas, todo ayuda a crear un sistema eficiente en costos.

Principios:

  1. Adoptar un modelo de consumo: Pagar solo por los recursos consumidos.
  2. Beneficiarse por la economía de escala: Al usar cloud computing se tiene acceso a recursos más poderosos a precios más bajos.
  3. Dejar de gastar dinero en centros de operaciones de información: AWS se encarga de la infraestructura.
  4. Análisis y atribución de gastos: Diferenciar entre los recursos usados.
  5. Uso de servicios para reducir el costo de mantenimiento: Rentar servicios elimina el mantenimiento.
Definición:
  1. Recursos económicos
  2. Igualar oferta y demanda
  3. Conciencia de gastos.
  4. Optimizar con el tiempo.
Recursos económicos: Usar el recurso ideal es la llave para la el bajo precio, fuera de la renta de recursos existen otros medio como la subasta de espacio en AWS EC2.

Igualar oferta y demanda. Tener los recursos necesarios para cumplir con la demanda sin pagar más de los que realmente se necesitan. En AWS se puede administrar todo esto automáticamente.

Conciencia de gastos. La automatización de AWS puede ser engañosa y hace fácil no darse cuenta de los recursos consumidos por lo que el monitoreo es necesario para comprobar gastos.

Optimizar con el tiempo. Buscar nuevas soluciones y mejores precios.

Excelencia Operacional.

Esto incluye como los cambios son ejecutados, y todas las respuestas a eventos no esperados. Todos estos cambios deben ser automatizados, documentados, probados y revisados regularmente.

Principios

  1. Ejecuta operaciones con código: Automatizar
  2. Alinea operaciones de proceso con objetivos empresariales: Colecta métricas que demuestren el cumplimiento de los objetivos empresariales.
  3. Haz cambios regulares, pequeños, e incrementales: El trabajo debe estar hecho de una manera para permitir cambios a los componentes de manera regular, se conoce como modularidad.
  4. Haz pruebas para eventos no esperados: Todo debe estar probado para encontrar las fallas y lidiar con ellas.
  5. Aprende de eventos y fallas: Logs y documentación regular.
  6. Mantén los procedimientos actualizados: La documentación tiene que ser actualizada con cada cambio y de manera que pueda ser entendido por cualquiera que vaya a ejecutar la operación.

Definición

  1. Preparación
  2. Operaciones
  3. Respuestas
Preparación: Es la idea de probar el ambiente y diversos escenarios, con AWS existen diversos métodos, servicios, y demás para garantizar pruebas. La documentación debe estar actualizada y probada para evitar errores y también para detectar.

Operaciones: Estandarizadas y manejadas de manera regular, todas las operaciones de rutina deben de ser automatizadas, las operaciones manuales deben de ser evitadas para eliminar la probabilidad de fallo, regresar a un estado pasado es más complicado después de un fallo así que se tiene que prevenir.

Respuestas: Las respuestas deben estar automatizadas y listas para responder a varios eventos.
